## 这个组织做什么

SakalioLabs 是一个 **AI-made project portfolio**。它不是传统公司主页，也不是空仓库集合，而是用来展示：

- Agent 能不能把一句想法做成一个可运行项目。
- Agent 能不能写清楚 README、演示地址、使用方式和项目边界。
- Agent 能不能持续迭代，而不是只生成一次性 demo。
- Agent 做出来的项目能不能被别人打开、理解、复用。

## 项目类型

| 展架 | 放什么 |
| --- | --- |
| Web Apps | 产品页、工具站、仪表盘、交互式前端、完整静态应用 |
| Agents | GitHub 自动化、文档维护、数据处理、任务执行工作流 |
| Games | AI 设计和实现的浏览器小游戏、模拟器、交互实验 |
| DevTools | CLI、脚手架、模板、检查器、代码生成辅助工具 |
| Data | 数据清洗、可视化界面、报表和分析型小工具 |
| Docs | Agent 维护的手册、教程、运行日志和项目规范 |

## 收录标准

每个放进来的项目，最好至少具备：

- **README:** 说明项目是什么、怎么运行、适合谁看。
- **Live Demo:** 如果能部署，就给一个可访问演示地址。
- **Source Code:** 代码结构清楚，能看到 Agent 的工作痕迹。
- **Status:** 标明是概念验证、可用版本、维护中还是归档。
- **Agent Notes:** 记录限制、设计选择和下一步可以继续让 Agent 做什么。
